# 表格规范说明

## 目的

用于统一项目内各类表格的展示规则，便于后续页面、文档和组件生成时保持一致性。

## 适用范围

- 前端页面中的数据表格
- 后台管理类列表表格
- 需求文档、设计文档中的结构化表格
- 后续新增的通用表格组件与配置

## 当前规范

### 1. 表头文案

- 英文表头统一使用 **Title Case**（各单词首字母大写）显示。
- 示例：
  - `project name` → `Project Name`
  - `owner email` → `Owner Email`
  - `created at` → `Created At`

## 扩展约定

- 本文件作为表格规范的统一维护入口。
- 后续如新增列宽、对齐方式、排序、空状态、操作列等要求，统一追加到本文件中。
- 以后新增或生成的表格，默认遵循本规范；如有特殊场景，需在对应需求或设计中单独说明。

## 维护建议

- 规范更新时，优先追加新条目，避免分散到多个文件。
- 如后续规范增多，可继续在当前目录下拆分为更细的专题文档。
